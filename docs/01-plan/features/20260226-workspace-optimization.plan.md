# [Plan] 워크스페이스 최적화 (Workspace Optimization)

- **작성일:** 2026-02-26
- **상태:** 계획 중
- **담당자:** Gemini CLI

## 1. 개요 (Overview)
프로젝트의 주요 아키텍처 리팩토링(모노리식 → 조립식 전략)이 대부분 완료됨에 따라, 더 이상 필요하지 않은 과거의 레거시 테스트 파일, 개발 전용 캐시, 로그 및 실험적 스크립트를 정리합니다. 이를 통해 워크스페이스의 순도를 높이고, 특히 LLM(Gemini CLI 등) 사용 시 불필요한 컨텍스트 토큰 소비를 획기적으로 줄이는 것을 목표로 합니다.

## 2. 분석 및 식별 (Analysis)
현재 프로젝트에서 정리가 필요한 항목들을 다음과 같이 분류합니다.

### A. 캐시 및 도구 데이터 (삭제 권장)
이들은 재생성 가능하거나 과거 작업의 상태를 담고 있어 현재 리팩토링된 구조에서는 혼란을 줄 수 있습니다.
- `.idea/`: IDE 설정 (로컬 환경 의존적)
- `.mypy_cache/`, `.pytest_cache/`, `.ruff_cache/`: 도구별 캐시
- `.omc/`, `.opencode/`, `.serena/`, `.sisyphus/`, `.taskmaster/`: 에이전트 도구들의 과거 상태 데이터

### B. 로그 및 임시 결과물 (삭제 권장)
- `analysis_log/`: 과거 분석 결과
- `htmlcov/`: 테스트 커버리지 리포트
- `logs/`: 과거 실행 로그
- `temp/`: 임시 파일

### C. 레거시 스크립트 및 테스트 (선별 삭제)
- `scripts/`: `test_binance_mainnet.py`, `simple_btc_stream.py` 등 초기 검증용 파일
- `tests/` (Root): `tests/` 디렉토리 루트에 있는 `test_ict_profiles.py`, `test_liquidation_config.py` 등. 이들은 현재 `src/` 산하의 모듈별 테스트 구조(예: `tests/strategies/`)와 중복되거나 과거 구조에 의존적일 가능성이 큽니다.

## 3. 목표 (Goals)
- **토큰 최적화:** LLM이 프로젝트 전체 구조를 파악할 때 읽어들이는 불필요한 파일 수 최소화.
- **아키텍처 명확화:** 신규 '조립식 전략' 아키텍처와 관련된 파일들만 남겨 개발 집중도 향상.

## 4. 실행 계획 (Tasks)
1. **[Plan]** 삭제 후보 리스트 확정 및 유저 승인 (현재 단계)
2. **[Design]** 안전한 삭제를 위한 백업 또는 체크리스트 설계
3. **[Do]** 파일 및 디렉토리 삭제 실행
4. **[Check]** 삭제 후 최신 아키텍처 기반 테스트(`tests/strategies/`, `tests/core/` 등) 정상 수행 확인

## 5. 기대 효과 (Expected Outcomes)
- LLM 컨텍스트 로딩 시간 및 비용 절감.
- 프로젝트 탐색 속도 향상.
- 개발 단계에서의 미세 조정 시 의도치 않은 레거시 파일 수정 방지.
